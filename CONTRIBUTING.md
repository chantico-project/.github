# Contributing to Chantico

Thank you for taking an interest in the Chantico open source project. We welcome
you to become involved in this project, whether it is by trying it out, using it
in your environment or also thinking along with our team to help improve the
data center management ecosystem.

## Prerequisites

### Code of Conduct
In your interaction with our project, it is essential to **adhere to our [Code of
Conduct](https://github.com/chantico-project/.github/blob/main/CODE_OF_CONDUCT.md)**.

### Developer Certificate of Origin
Contributors will be asked to undersign a Developer Certificate of Origin (DCO)
before they submit the first pull request to this project. We use [the standard
DCO text of the Linux Foundation](https://developercertificate.org/).

### License
Your contributions to Chantico must be licensed properly:

- Code contributions must be licensed under the [Apache 2.0
   License](http://www.apache.org/licenses/LICENSE-2.0).
- Documentation contributions must be licensed under [CC-BY-SA
  4.0](https://creativecommons.org/licenses/by-sa/4.0/deed.en).

## Contributing

### Issues

We welcome issues to track bugs and enhancement requests that are in line with
the scope and vision of the project. Please provide as much context as possible
when you open an issue. The information you provide must be comprehensive enough
to understand, reproduce the behavior and find related reports of that issue for
the assignee. A template is provided as a suggestion for a good issue.

### Discussions

Discussions may be used for broader topics, such as support and sharing 
of setups, or figuring out whether certain (bigger) changes are welcome.

### Pull Requests

We welcome source code contributions through pull requests, but please know that
**not all proposed contributions can be accepted**. Some features may fit
another project better or don't fit the general direction of this project. It is
suggested to discuss first and implement later, especially for larger changes.

Please use the following steps for a succesful pull request:

1. Make sure the change is welcome through an issue or discussion.
2. Create a branch by forking the repository and make your changes.
3. Commit and push your changes on that branch, make sure to include the sign
   off line in the commit message.
4. Ensure your change passes local testing and has added tests where appropriate.
5. Create a pull request and complete the suggested template.
6. For the first pull request: follow the instructions to sign the Developer
   Certificate of Origin.
7. Engage with us in code review and potential suggestion for improvement.
8. Celebrate once the pull request has been approved and merged!
